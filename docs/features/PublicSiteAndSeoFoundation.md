# Public Site and SEO Foundation

## Goal

Provide a polished public-facing school website in `apps/www` that feels like a real admissions site, stays mobile-first, and can be rebranded later for other tenants without rewriting the page structure.

## Scope Boundary

This feature covers the public marketing surface only:

- homepage
- about page
- academics page
- admissions page
- fees page
- visit page
- contact page
- metadata, robots, sitemap, and structured data

It does **not** include:

- admissions workflow automation
- enquiry form backends
- CRM handoff logic
- marketing-media automation
- admin/teacher/portal changes

## Data Flow

1. The app layout injects a school theme through CSS variables so the surface can be rebranded later.
2. Shared site copy, navigation, metadata helpers, and page content arrays live in `apps/www/lib/site.ts`.
3. The homepage renders a structured data block for search engines and provides the core conversion path for prospective families.
4. Supporting pages expand the school story across about, academics, admissions, fees, visit, and contact paths.
5. `robots.ts` and `sitemap.ts` expose a clean public crawl surface with canonical URLs.

## Implemented Outcome

- `apps/www` now renders as a real public website surface.
- The homepage communicates the school story, trust signals, academic phases, and admissions entry points.
- Supporting marketing pages cover about, academics, admissions, fees, visit, and contact content.
- Metadata is in place for titles, descriptions, canonical URLs, open graph data, and Twitter cards.
- `robots.txt`, `sitemap.xml`, and a simple favicon are generated by Next.js.
- The content model stays reusable because school branding, contact details, and copy live in one shared configuration file.

## Regression Checks

- The public site must remain separate from admin, teacher, and portal surfaces.
- Navigation must stay mobile-friendly and easy to scan.
- Copy should stay school-facing rather than product-facing.
- Future tenant branding should only require replacing the shared site config, not rebuilding the page tree.
