# T12 Platform Pricing and Packaging Strategy

## Objective

Reconstruct the platform's commercial strategy so the SaaS can be sold to schools with a clear separation between platform subscription billing and school fee billing.

## Requested Scope

- define the platform pricing model as setup + recurring + optional upgrades
- segment the offer into practical package tiers for different school sizes and service needs
- define entitlement categories separately from plan names
- keep the commercial strategy separate from school fee billing and Paystack routing
- keep the package model separate from public-web operating mode decisions
- make the output ready for later platform billing implementation and T17 messaging

## Acceptance Criteria

- [x] Platform SaaS billing is explicitly separated from school billing.
- [x] The strategy uses a mixed model with setup fees, recurring fees, and optional upgrades.
- [x] Package tiers are segmented for small private schools, mid-tier schools, and premium schools.
- [x] Entitlement categories are defined separately from tier names.
- [x] Public-web operating mode remains independent from package tier choice.
- [x] The strategy is implementation-ready for later platform billing work and the T17 marketing site.

## Notes

- This is a docs-only strategy task.
- Do not introduce school-fee billing detail into this package strategy.
- Exact currency amounts can be decided later; this task fixes the commercial structure, names, and entitlement boundaries now.
