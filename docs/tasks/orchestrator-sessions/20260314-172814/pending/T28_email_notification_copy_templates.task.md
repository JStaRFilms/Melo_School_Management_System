# T28 Email and Notification Copy Templates

**Mode:** `vibe-code`  
**Workflow:** `/vibe-build`

## Agent Setup (DO THIS FIRST)

- Read `/vibe-build`.
- Run `/vibe-primeAgent`.
- Load `copywriting`.
- Do not use `context7`.

## Objective

Write reusable copy and template content for onboarding, academic, and billing notifications across email and in-app messaging.

## Scope

Included: invite messaging, result-ready notices, invoice/payment notices, tone guidelines.  
Excluded: message delivery infrastructure itself.

## Context

These templates will feed the notification feature and should support school branding tokens.

## Definition of Done

- Core notification templates are drafted.
- Tone and placeholders are consistent.
- Billing and academic events are both covered.

## Expected Artifacts

- copy/template docs and any email-template source files introduced

## Constraints

- Keep language concise and parent-friendly.
- Do not hardcode one school’s identity unless tokenized.

## Verification

- Confirm invite, report publication, invoice issued, payment received, and payment failed flows have template content.
