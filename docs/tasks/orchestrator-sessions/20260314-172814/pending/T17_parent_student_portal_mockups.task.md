# T17 Parent and Student Portal Mockups

**Mode:** `vibe-architect`  
**Workflow:** `/vibe-design`

## Agent Setup (DO THIS FIRST)

- Read `/vibe-design`.
- Run `/vibe-primeAgent`.
- Load `frontend-design` and `ui-ux-pro-max`.
- Do not use `context7`.

## Objective

Design the parent and student portal screens for dashboards, report cards, payment views, notifications, and linked-student switching.

## Scope

Included: parent and student landing pages, academic views, billing views, alerts.  
Excluded: admin and teacher workflows.

## Context

Portal users need confidence, simplicity, and clear access to academic and billing status.

## Definition of Done

- Portal mockups include academic and billing journeys.
- Parent and student variations are represented where behavior differs.

## Expected Artifacts

- Portal HTML mockups in `docs/mockups/`

## Constraints

- Keep sensitive data readable and uncluttered on mobile.
- Parents must not be forced through student-style navigation.

## Verification

- Confirm report-card views, invoice views, and payment history screens are included.
