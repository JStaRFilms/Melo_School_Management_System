## Objective

Design the admin app flows for onboarding, school configuration, academics setup, results oversight, support panels, and billing operations.

## Scope

Included: admin dashboards, list/detail flows, forms, support panels.  
Excluded: teacher or portal screens.

## Context

This app owns the heaviest operational workflows and must remain usable on mobile.

## Definition of Done

- Admin mockups cover onboarding, academics, results, and billing.
- The navigation and page hierarchy are coherent.

## Expected Artifacts

- Admin HTML mockups in `docs/mockups/`

## Constraints

- Keep support panels read-only in the design.
- Avoid desktop-only data-table assumptions.

## Verification

- Confirm billing and collections views are represented.
- Confirm admin onboarding flows are included.
