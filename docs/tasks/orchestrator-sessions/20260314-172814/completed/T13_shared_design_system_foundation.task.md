# T13 Shared Design System Foundation

**Mode:** `vibe-architect`  
**Workflow:** `/vibe-design`

## Agent Setup (DO THIS FIRST)

- Read `/vibe-design`.
- Run `/vibe-primeAgent`.
- Load `frontend-design` and `ui-ux-pro-max`.
- Do not use `context7`.

## Objective

Produce the shared design-system foundation that all apps will implement, including tokens, typography, core components, form patterns, navigation, and responsive rules.

## Scope

Included: token system, component states, layout rules, navigation patterns.  
Excluded: app-specific page mockups.

## Context

This is the base visual contract for the monorepo UI packages and all page mockups.

## Definition of Done

- A portable design-system artifact exists.
- Tokens and core components are defined clearly enough to implement later.

## Expected Artifacts

- `docs/design/design-system.html`

## Constraints

- Keep the system mobile-first and accessible.
- Avoid over-complicated component variants that the first release does not need.

## Verification

- Confirm buttons, inputs, cards, nav, tables, and print-friendly styles are represented.
