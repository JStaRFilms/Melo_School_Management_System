# [FR-004] Academic Structure Management

## Labels
`MUS`, `academics`, `admin`

## User Story
As a school admin, I want to define sessions, terms, classes, subjects, and teacher assignments, so that both primary and secondary workflows are supported.

## Proposed Solution

### Overview
Create school-scoped academic records that support class teachers handling multiple subjects and subject-specialist teachers handling specific class-subject combinations.

### Implementation Flow
1. Create session, term, class, and subject entities.
2. Add class-subject-teacher assignment records.
3. Add validation rules for primary and secondary assignment modes.

### Key Considerations
- The same model must support primary and secondary school operations.
- Term and year boundaries affect results and billing.

## Acceptance Criteria

- [ ] Admins can define academic years, terms, classes, and subjects.
- [ ] Teacher assignments support both primary and secondary patterns.
- [ ] Academic records remain scoped to one school.
