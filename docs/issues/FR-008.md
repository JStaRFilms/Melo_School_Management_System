# [FR-008] Branded Printable Report Cards

## Labels
`MUS`, `reports`, `printing`

## User Story
As a school, I want printable and portal-ready report cards with comments and branding, so that result delivery is professional and consistent.

## Proposed Solution

### Overview
Generate report-card payloads from published results, merge them with school branding and comment fields, and expose printable layouts plus portal views.

### Implementation Flow
1. Build report-card view models.
2. Add teacher and head-teacher comment support.
3. Render print-friendly report-card pages.

### Key Considerations
- Report cards must remain readable on paper and on screen.
- Branding should be school-specific without hardcoding a single school.

## Acceptance Criteria

- [ ] Report cards include scores, comments, and branding.
- [ ] Print layouts are usable for term-end distribution.
- [ ] Published report cards are viewable in the portal.
- [x] Missing cumulative annual data surfaces a direct admin backfill link from report-card review and blocked print flows.
