# [FR-002] Role-Based Authentication and Routing

## Labels
`MUS`, `auth`, `roles`

## User Story
As a user, I want to sign in once and land in the correct workspace for my role and school, so that I can start work without confusion.

## Proposed Solution

### Overview
Use Better Auth with unified people/accounts and school memberships, then derive role-aware entry points for admin, teacher, parent, and student experiences.

### Implementation Flow
1. Define account and membership models.
2. Implement sign-in, session resolution, and school selection.
3. Route users to the correct app and landing page.

### Key Considerations
- One person may have multiple roles.
- Default routing must stay deterministic and overridable.

## Acceptance Criteria

- [ ] Users can authenticate successfully.
- [ ] Membership determines accessible schools and roles.
- [ ] Post-login routing lands users in the correct workspace.
