# [FR-009] Parent and Student Academic Portal

## Labels
`MUS`, `portal`, `academics`

## User Story
As a parent or student, I want to view results, report cards, and academic notifications online, so that I can track performance without visiting school.

## Proposed Solution

### Overview
Create a role-aware portal with dashboards, report views, result history, and school-scoped notifications.

### Implementation Flow
1. Build parent and student dashboard routes.
2. Connect report-card and result-history data.
3. Add notice and notification surfaces.

### Key Considerations
- Parents need linked-student selection where relevant.
- Students should only see their own academic records.
- Parents may eventually have children linked across more than one school, so portal routing cannot assume one parent equals one school forever.

## Future Extension Note

- Keep `portal.meloschool.com` as the canonical authenticated host even when multiple schools exist.
- After sign-in, if a parent has more than one valid child/school context, show a chooser such as "Which child do you want to manage?" before loading the dashboard.
- School branding inside the portal should come from the selected child/school context so the dashboard can feel school-specific without needing school-specific portal URLs.

## Acceptance Criteria

- [x] Parents can view linked student results and report cards.
- [x] Students can view their own academic records.
- [x] Portal notifications surface key academic events.
- [x] Current-school admins can provision or reset temporary Better Auth portal credentials for students and linked parents from the admin roster and student-first onboarding flow.
- [ ] Parents linked to children across multiple schools can choose an active child/school context before entering the portal dashboard.
