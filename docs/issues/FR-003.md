# [FR-003] School Administration Onboarding

## Labels
`MUS`, `admin`, `onboarding`

## User Story
As a school admin, I want to onboard admins, teachers, students, and parents, so that the school can start using the system end to end.

## Proposed Solution

### Overview
Provide admin workflows to create or invite users, assign roles, and capture school-specific profile data needed for academics and billing.

### Implementation Flow
1. Build people and membership creation flows.
2. Add invite or activation handling where appropriate.
3. Support bulk-safe admin actions for school setup.

### Key Considerations
- Onboarding should work for one school without breaking future multi-school support.
- Parent-student relationships must be validated.

## Acceptance Criteria

- [~] Admins can create or invite staff and families.
- [~] Teachers and students are assignable to one school; parent onboarding is still pending.
- [~] Required onboarding data is captured for current academic setup flows, but family-facing onboarding is not complete yet.

## Current Alignment Snapshot

What is already aligned:

- school admins can create teachers
- school admins can create classes, students, subjects, sessions, and terms
- student academic enrollment flows now exist
- platform super admin can provision the first school admin for a new school

What is not complete yet:

- parent onboarding and parent-student linking
- school-admin-to-sub-admin hierarchy and recovery flow
- invite-style onboarding flows instead of direct credential provisioning everywhere
- dedicated student onboarding page requested in the dev log

Recommended split for future work:

- keep `teacher/student/class/session/subject` onboarding in the school admin slice
- keep `first school admin creation` in the platform slice
- add `sub-admin creation and succession` as a dedicated school-admin follow-up
