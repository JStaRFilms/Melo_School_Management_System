# [FR-003] School Administration Onboarding

## Labels
`MUS`, `admin`, `onboarding`

## User Story
As a school admin, I want to onboard admins, teachers, students, and parents, so that the school can start using the system end to end.

## Proposed Solution

### Overview
Provide admin workflows to create or invite users, assign roles, and capture school-specific profile data needed for academics and billing.

### Implementation Flow
1. Build people and membership creation flows.
2. Add invite or activation handling where appropriate.
3. Support bulk-safe admin actions for school setup.

### Key Considerations
- Onboarding should work for one school without breaking future multi-school support.
- Parent-student relationships must be validated.

## Acceptance Criteria

- [ ] Admins can create or invite staff and families.
- [ ] Teachers, students, and parents are assignable to one school.
- [ ] Required onboarding data is captured and validated.
