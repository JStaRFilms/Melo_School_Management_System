# [FR-014] Manual and Online Payment Tracking

## Labels
`MUS`, `payments`, `billing`

## User Story
As a school admin or parent, I want both offline and online payments captured against invoices, so that collections and balances stay accurate.

## Proposed Solution

### Overview
Support manual cash or bank entries plus online checkout via a provider-agnostic payment layer optimized for Paystack first.

### Implementation Flow
1. Add payment-record and allocation models.
2. Implement payment initialization and verification.
3. Support manual entry and reconciliation flows.

### Key Considerations
- Payment references must be unique and auditable.
- Gateway callbacks must be signature-verified before any state change.

## Acceptance Criteria

- [ ] Manual payments can be recorded against invoices.
- [ ] Online payments can be initialized and verified.
- [ ] Successful payments update invoice balances safely.
