# [FR-007] Results Entry and Moderation

## Labels
`MUS`, `results`, `workflow`

## User Story
As a teacher or admin, I want controlled score entry and editing with audit fields, so that saved results are accurate and traceable.

## Proposed Solution

### Overview
The current v1 ships draft exam-entry sheets with role-based access controls, bulk save, and audit metadata. Approval and publication states are deferred.

### Implementation Flow
1. Load the exam-entry sheet for the selected session, term, class, and subject.
2. Enforce school membership plus teacher assignment or admin access.
3. Save bulk rows with `enteredBy`, `updatedBy`, timestamps, and draft status.
4. Preserve raw and derived values for future moderation and regrading.

### Key Considerations
- Teachers can only edit assigned sheets; admins can edit any sheet in their school.
- Draft-only rows are stored today; submitted, approved, and published states are future work.

## Acceptance Criteria

- [x] Teachers and admins can enter and edit draft exam-entry sheets within their school.
- [x] Result rows store `enteredBy`, `updatedBy`, `createdAt`, and `updatedAt`.
- [x] Cross-school and unauthorized sheet access are blocked.
- [ ] Submission, approval, and publication workflow states remain future scope.
