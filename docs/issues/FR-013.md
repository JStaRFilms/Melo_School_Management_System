# [FR-013] School Fee Plans and Invoicing

## Labels
`MUS`, `billing`, `fees`

## User Story
As a bursar or admin, I want to define fee plans, invoices, waivers, and installment rules, so that every student has a clear payable balance.

## Proposed Solution

### Overview
Create school-aware fee plans that generate student invoices with line items, discounts, waivers, due dates, and installment policies.

### Implementation Flow
1. Define fee-plan and invoice schemas.
2. Add invoice generation rules by class, term, and school.
3. Add balance computation and status tracking.

### Key Considerations
- Invoice state must stay correct after partial payments.
- Fee plans should be reusable across class groups and terms.

## Acceptance Criteria

- [x] Admins can define fee plans and installment policies.
- [x] Student invoices can be generated from fee plans.
- [x] Balances, waivers, and invoice status are computed correctly.
