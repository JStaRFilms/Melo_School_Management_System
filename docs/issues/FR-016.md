# [FR-016] AI-Assisted Teacher Tools

## Labels
`MUS`, `ai`, `teacher`, `planning`

## User Story
As a teacher, I want OCR for handwritten lesson notes and AI-generated quizzes and CBTs, so that preparation work is faster and more structured.

## Proposed Solution

### Overview
Keep AI generation grounded in school-scoped materials, but move orchestration to a context-first model. AI generation should be launched from topic context for lesson and quiz work, and from subject-level exam scope for exam drafting, while still allowing teachers to add supporting materials from the library within the workflow.

### Implementation Flow
1. Resolve planning context before generation starts.
2. Suggest topic-bound and broad planning materials inside the chosen context.
3. Allow teachers to add or remove materials without leaving the current planning flow.
4. Keep generation editable, logged, rate-limited, and human-reviewed before any student-facing use.
5. Preserve compatibility for existing drafts and broad planning references.

### Technical Approach
- Treat selected sources as grounding inputs, not the sole identity of a draft.
- Preserve existing Convex audit, approval, and rate-limit patterns.
- Keep exam-draft generation explicitly wider in scope than topic-based quiz generation.

### Key Considerations
- AI output must remain editable before student-facing use.
- Costly AI actions should be rate-limited and logged.
- Assessment generation should support school-defined profiles plus bounded teacher overrides for question mix and open-ended direction.
- Broad planning references must remain reusable without being misclassified as topic resources.

## Acceptance Criteria

- [x] Teachers can upload note material for OCR processing.
- [x] Image-heavy/scanned planning-library PDFs can fall back to OpenRouter multimodal OCR when `OPENROUTER_API_KEY` is configured.
- [ ] OCR output becomes editable lesson content.
- [x] Teachers can generate quizzes or CBT drafts from the processed material.
- [ ] Lesson-plan generation is launched from explicit topic context or an explicit compatibility fallback.
- [ ] Topic-level quiz and class-test generation stay topic-scoped by default.
- [ ] Exam-draft generation uses explicit exam scope rather than pretending to be topic-bound.
- [ ] Existing broad planning references continue to support generation without forced topic attachment.
- [ ] Existing source-first drafts continue to open safely during rollout.

## Current Alignment Snapshot

What is already aligned:

- AI generation already exists for lesson plans and question-bank flows
- broad planning references already work through `imported_curriculum` handling
- current workspaces already require topic input when sources are too broad to infer a topic
- AI run metadata, access checks, and source selection persistence already exist

What is missing for full FR-016 completion in this lane:

- context-first launch before AI generation
- topic vs exam scope separation in generation entry points
- context-aware source suggestion and add-material flow inside the workspace
- compatibility-mode handling for legacy source-first drafts during the transition

## Implementation Notes

- Assessment authoring already supports admin-defined assessment generation profiles and teacher-side bounded question-mix overrides.
- Effective generation settings should continue to persist with saved assessment drafts and AI run metadata after the v2 context refactor.
- Approval, visibility, and portal exposure rules from the existing knowledge-hub implementation remain in force.
- 2026-04-28: Lesson-plan, student-note, and assignment generation now require a resolved instruction template before calling the model. The model generates template-bound sections by exact section id/label, required section coverage is validated, and drafts render directly from the resolved template order instead of a generic schema or fixed label mapper.
- 2026-04-30: Topic workspace source selections now sync between the lesson-plan side and topic assessment/question-bank side in the same browser session. Practice quizzes and class tests opened for the same class, term, subject, level, and topic hydrate the existing lesson workspace source selection when no explicit sourceIds are present in the URL.
- 2026-04-30: Assessment profile question-style changes now apply matching generation distribution presets before save, so objective-heavy/open-ended-heavy choices change the persisted `questionMix` used by future assessment generation. Inactive profiles are treated as archived: the admin profile UI has explicit Archive/Restore actions, and the backend prevents archived profiles from remaining school defaults.
- 2026-05-01: Planning-library PDF extraction now uses native PDF text parsing first, then OpenRouter PDF fallback with `google/gemma-4-31b-it:free` via `OPENROUTER_API_KEY` and the `file-parser` plugin using the free `cloudflare-ai` engine. Selected-page uploads process the trimmed selected-pages PDF rather than the full original document, and OCR fallback chunks retain selected source-page metadata.
