# [FR-006] Assessment and Grading Engine

## Labels
`MUS`, `results`, `grading`

## User Story
As a teacher, I want to enter three CA scores and an exam score with school-defined grading rules, so that totals, grade letters, and remarks are calculated automatically.

## Proposed Solution

### Overview
Ship the exam-recording v1 calculation engine as a bulk-entry workflow. Teachers and admins select a session, term, class, and subject, then enter raw scores while the system computes the final result from school-defined assessment settings and grading bands.

### Implementation Flow
1. Load the roster, current scores, active school assessment settings, and grading bands.
2. Validate CA and exam ranges for the active school mode.
3. Convert exam input according to the school-wide exam rule.
4. Derive the total, grade letter, and remark before persistence.

### Key Considerations
- Calculations must be repeatable and auditable.
- The school-wide `/60 -> /40` rule is supported for primary-school flows.
- Ranking, averages, and CGPA are deferred to a future slice.

## Acceptance Criteria

- [x] Teachers can store `CA1`, `CA2`, `CA3`, and exam scores per subject and student.
- [x] Totals, grade letters, and remarks are computed automatically.
- [x] Grading behavior is configurable per school, including `raw40` and `raw60_scaled_to_40`.
- [ ] Averages, ranking, and CGPA remain future scope.
