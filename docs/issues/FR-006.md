# [FR-006] Assessment and Grading Engine

## Labels
`MUS`, `results`, `grading`

## User Story
As a teacher, I want to enter three CA scores and an exam score with school-defined grading rules, so that averages, rankings, and CGPA are calculated automatically.

## Proposed Solution

### Overview
Build an assessment scheme model with configurable components, total-score rules, grading thresholds, ranking behavior, and aggregate calculations.

### Implementation Flow
1. Define assessment scheme and score tables.
2. Implement calculation services for totals, averages, ranking, and CGPA.
3. Recompute derived values when scores or schemes change.

### Key Considerations
- Calculations must be repeatable and auditable.
- Ranking may be school-specific and should be configurable.

## Acceptance Criteria

- [ ] Teachers can store CA and exam scores per subject and student.
- [ ] Totals, averages, ranking, and CGPA are computed automatically.
- [ ] Grading behavior is configurable per school.
