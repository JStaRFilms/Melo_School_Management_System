# [FR-011] Public Website and Admissions-Ready Content

## Labels
`MUS`, `website`, `marketing`, `seo`

## User Story
As a prospective family, I want a polished school website with clear information and calls to action, so that I can learn about the school and contact it easily.

## Proposed Solution

### Overview
Build a reusable public-site foundation in `apps/www` with school branding, polished marketing copy, and SEO-friendly page structure.

### Implementation Flow
1. Centralize school brand, navigation, and page metadata in a shared config.
2. Build the homepage plus essential supporting public pages from the sitemap.
3. Add SEO baseline support with canonical metadata, robots, sitemap, and structured data.

### Key Considerations
- The design must feel premium and intentional, not generic.
- The site must stay reusable for later schools through one shared brand/config layer.
- Admissions should stay informational; no workflow automation is required for this FR.

## Acceptance Criteria

- [x] The public site supports school-specific branding and copy.
- [x] The homepage is implemented and mobile-first.
- [x] Essential supporting pages are implemented from the sitemap: About, Academics, Admissions, Fees, Visit, and Contact.
- [x] Public CTAs and contact intent flows are present.
- [x] Metadata, canonical URLs, robots, sitemap, and basic structured data are in place.
- [x] The implementation remains isolated to the public `www` surface.
