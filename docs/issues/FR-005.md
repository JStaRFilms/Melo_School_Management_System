# [FR-005] Student Enrollment and Parent Linking

## Labels
`MUS`, `students`, `families`, `admin`

## User Story
As a school admin, I want students linked to classes, subjects, and parents, so that learning and billing data reflect the right families.

## Proposed Solution

### Overview
Model student enrollments separately from person accounts so class placement, subject subscriptions, and parent relationships are auditable and term-aware.

### Implementation Flow
1. Create enrollment records for class and subject membership.
2. Create parent-student relationship records.
3. Expose enrollment data to academic and billing flows.

### Key Considerations
- Students may move classes between sessions or terms.
- A student may map to one or more guardians.

## Implementation Progress

- Student/class/subject enrollment remains on the existing roster workflow.
- Parent identity and family links now create school-scoped family records from the student profile editor.
- Billing and portal surfaces now consume the same family records.
- Household management now supports duplicate-email review, parent contact maintenance, parent removal guardrails, and clearer student-vs-household destructive actions.

## Acceptance Criteria

- [x] Students can be assigned to classes and subjects.
- [x] Parents can be linked to one or more students.
- [x] Enrollment data is reusable by results and billing features.
