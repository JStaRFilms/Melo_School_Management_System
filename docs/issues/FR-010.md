# [FR-010] Teacher Workspace

## Labels
`MUS`, `teacher`, `workspace`, `planning`

## User Story
As a teacher, I want a dashboard for lesson planning, class work, results, and assessment tools, so that my daily workflow lives in one place.

## Proposed Solution

### Overview
Keep the teacher app as the role-specific home for planning and delivery work, but make planning context-first instead of source-first. For the Lesson Knowledge Hub v2 slice, teachers should launch planning from subject/class/term context, then open a topic workspace or exam workspace with source suggestions and add-material actions available inside that context.

### Implementation Flow
1. Introduce a planning hub or launcher that becomes the default planning entry.
2. Let teachers choose `Subject -> Level/Class -> Term -> Topic` for lesson and quiz work.
3. Let teachers choose `Subject -> Level/Class -> Term -> Exam scope` for exam drafting.
4. Preserve library access as a repository surface, not the required first step for authoring.
5. Keep recent drafts and resume behavior tied to planning context.

### Technical Approach
- Reuse the existing planning routes where practical, but move routing intent from raw `sourceIds` toward explicit context descriptors.
- Keep teacher authorization limited to assigned academic scope and school context.
- Separate launch, repository, and authoring responsibilities so the workspace does not need to double as a library admin surface.

### Key Considerations
- Teacher UX should prioritize speed on mobile devices.
- Access should remain limited to assigned classes and subjects.
- Existing source-first links and drafts must continue to open safely during migration.
- Topic work and exam work need visibly different framing to prevent scope confusion.

## Acceptance Criteria

- [x] Teachers have a dedicated workspace with role-specific navigation.
- [ ] Teachers can access assigned classes and subjects quickly.
- [x] Workspace exposes planning and result-entry flows.
- [ ] Planning launches from a teacher-facing context selector instead of defaulting to library source selection.
- [ ] Topic-first planning is available for lesson plans, student notes, assignments, and topic-level quizzes.
- [ ] Exam drafting is available from a subject-scope exam flow distinct from topic work.
- [ ] Recent and existing drafts can be resumed without losing context.
