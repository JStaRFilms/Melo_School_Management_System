# [FR-015] Billing and Collections Dashboards

## Labels
`MUS`, `billing`, `admin-ui`

## User Story
As an admin, I want visibility into outstanding balances, paid invoices, reconciliation, and collection status, so that finance operations are manageable.

## Proposed Solution

### Overview
Build admin-facing billing dashboards and collection views backed by invoice, payment, and balance aggregates.

### Implementation Flow
1. Create finance summary queries.
2. Build invoice list, payment history, and collection state views.
3. Add filtering for class, term, status, and payer.

### Key Considerations
- Dashboard performance matters for large student lists.
- Finance surfaces must remain school-scoped and permission-aware.

## Acceptance Criteria

- [x] Admins can view invoices, balances, and payments in one workspace.
- [x] Collections can be filtered by class, term, and status.
- [x] Reconciliation information is visible for manual and gateway payments.

## Alignment Note

This dashboard issue belongs in the school admin app, not the platform app.

Platform dashboards may later show:

- school subscription health
- number of overdue schools
- trial vs active schools

But this issue specifically refers to the school's own collections dashboard.
