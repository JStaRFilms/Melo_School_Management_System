# [FR-018] Automated Verification and End-to-End Confidence

## Labels
`MUS`, `testing`, `quality`

## User Story
As a delivery team, I want repeatable lint, typecheck, build, and Cypress coverage, so that releases are stable.

## Proposed Solution

### Overview
Create a verification harness that standardizes type checks, linting, builds, and end-to-end flows across the monorepo.

### Implementation Flow
1. Define root scripts and turbo pipelines.
2. Add Cypress critical-path coverage.
3. Wire the verification flow into handoff tasks.

### Key Considerations
- E2E scope should focus on business-critical flows.
- The repo needs repeatable commands from root.
- Current automated browser coverage is Playwright-based; the historical Cypress wording in this issue should be interpreted as end-to-end coverage in general, not a hard dependency on Cypress specifically.

## Current Verification Baseline

Last verified: `2026-04-19`

- `pnpm build` ✅ passes across the monorepo.
- `pnpm typecheck` ✅ passes across the monorepo.
- `pnpm lint` ✅ passes across the monorepo.
- `pnpm test` ✅ passes across the monorepo.
  - `@school/shared`, `@school/convex`, `@school/admin`, and `@school/teacher` all run real Vitest suites.
  - `@school/portal` currently has no test files and now uses `vitest run --passWithNoTests` so the release gate reflects reality without failing spuriously.
- `pnpm test:e2e` ✅ runs Playwright smoke coverage.
- The current smoke suite seeds demo academic data, launches the admin and teacher apps automatically, verifies admin assessment setup routes, and verifies teacher live exam-entry roster loading.
- E2E breadth is still limited to academic smoke coverage; broader school setup, results, billing, portal, and public-web paths remain future work.

## Acceptance Criteria
